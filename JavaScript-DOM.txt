In HTML, each tag in HTML is a node:
    + The entire document is a document node
    + Every HTML element is an element node
    + The text inside HTML elements are text nodes
    + Every HTML attribute is an attribute node (deprecated)
    + All comments are comment nodes 
--> With the HTML DOM, all nodes in the node tree can be accessed by JavaSCript
--> New node can be created, and all nodes can be modifired or deleted

1. Node Relationships
- The nodes in the node tree have a hirarachcal relationship to each other
- The terms parent, child, and sibling are used to describe the relationships:
    + In a node tree, the top node is called the root (root node)
    + Every node has exactly one parent, except the root (which has no parent)
    + A node can have a number of children
    + Siblings are nodes with the same parent 

2. Navigating between nodes
- Use these following properties to nagigate nodes with JavaSCript:
    + parentNode
    + childNodes[node_number]
    + firstChild
    + lastChild
    + nextSibling
    + previousSibling 

*** Note:
    --------------------------------------------
    |   <title id="demo">DOM Tutorial</title>   |
    --------------------------------------------
 - We treat 'DOM Tutorial' as a text node not a text attribute in 'demo'
 - So we have many way to access this text node:
    + Use innerHTML --> myTitle = document.getElementById('demo').innerHTML;
    + Using first child and 'nodeValue - property' --> myTitle = document.getElementById('demo').firstChild.nodeValue;
    + Access by node index --> myTitle = document.getElementById('demo').childNodes[0].nodeValue;

DOM Root Nodes 
- There are two special properties that allow access to the full document:
    + document.body - The body of full document
    + document.documentElement - The full document 

*** document.body 
- Represents the <body> element
- Refers only to the content area of the page 
- Contains everything visible to the user:
    + Text
    + Images
    + Buttons
    + Forms
    + Scripts rendered on the page 
- Common uses:
    + Manipulating page content
    + Changing background color
    + Adding event listeners

*** document.documentElement 
- Represents the <html> element (the root of the document)
- Refers to the entire document
- Contains:
    + <head>
    + <body> 
- Common uses:
    + Getting full page dimensions 
    + Handling scrolling
    + Working with CSS variables 

3. The nodeName Property 
- The nodeName property specifies the name of a node:
    + nodeName is read-only 
    + nodeName of an element node is the same as the tag name
    + nodeName of an attribute node is the attribute name
    + nodeName of a text node is always #text 
    + nodeName of the document node is always #document 

4. The nodeValue Property
- The nodeValue property specifies the value of a node:
    + nodeValue for element nodes is null 
    + nodeValue for text nodes is the text itself 
    + nodeValue for attribute node is the attribute value

5. The nodeType Property
- The nodeType property is read only. It returns the type of a node

JavaSCript HTML DOM Elements (Nodes)

1. Creating new HTML Elements (Nodes) 
- To add a new element to the HTML DOM, you must create the element (element node) first, and then append it to an existing element 
- Example:
 <div id="div1">
  <p id="p1">This is a paragraph.</p>
  <p id="p2">This is another paragraph.</p>
</div>

<script>
const para = document.createElement("p");
const node = document.createTextNode("This is new.");
para.appendChild(node);

const element = document.getElementById("div1");
element.appendChild(para);
</script> 

2. Creating new HTML Elements - insertBefore()
- The appendChild() method appended a new element as the last child of the parent 
- If you dont want that you can use 'insertBefore()' method
- Example:
 <div id="div1">
  <p id="p1">This is a paragraph.</p>
  <p id="p2">This is another paragraph.</p>
</div>

<script>
const para = document.createElement("p");
const node = document.createTextNode("This is new.");
para.appendChild(node);

const element = document.getElementById("div1");
const child = document.getElementById("p1");
element.insertBefore(para, child);
</script> 

3. Removing Existing HTML Elements 
- To remove an HTML element, use the 'remove()' method:
- Example:
 <div>
  <p id="p1">This is a paragraph.</p>
  <p id="p2">This is another paragraph.</p>
</div>

<script>
const elmnt = document.getElementById("p1"); elmnt.remove();
</script> 

4. Removing a Child Node
- Example:
 <div id="div1">
  <p id="p1">This is a paragraph.</p>
  <p id="p2">This is another paragraph.</p>
</div>

<script>
const parent = document.getElementById("div1");
const child = document.getElementById("p1");
parent.removeChild(child);
</script> 

5. Replacing HTML Elements
- To replace an elemen to the HTML DOM, use the 'replaceChild()' method:

<div id="div1">
  <p id="p1">This is a paragraph.</p>
  <p id="p2">This is another paragraph.</p>
</div>

<script>
const para = document.createElement("p");
const node = document.createTextNode("This is new.");
para.appendChild(node);

const parent = document.getElementById("div1");
const child = document.getElementById("p1");
parent.replaceChild(para, child);
</script>

JavaSCript HTML DOM Collections 

1. The HTMLCollection Object 
- The getElementByTagName() method returns an HTMLCollection object
- An HTMLCollection object is an array-like list of HTML elements
- Example:
const myCollection = document.getElementsByTagName("p");
--> The element in the collection can be accessed by an index number 
--> The index starts at 0 

*** Note
- An HTMLCollection is NOT an array
- HTMLCollection can access by index as an array, but it cannot fully use method of array like: pop(), push(), ...

JavaSCript HTML DOM Node Lists 

1. The HTML DOM NodeList Object 
- A NodeList object is a list of nodes extracted from a document
- A NodeList object is almost the same as an HTMLCollection
- All browser return a NodeList object for the property 'childNodes'
- Most browser return a NodeList object for the method 'querySelectorAll()'
+---------------------------+-------------------------------+-------------------------------+
| Feature                   | HTMLCollection                | NodeList                      |
+---------------------------+-------------------------------+-------------------------------+
| Type of collection        | HTML elements only            | DOM nodes                     |
| Includes text nodes       | No                            | Yes (text, comments)          |
| Live (auto-updating)      | Always live                   | Sometimes (usually static)    |
| Updates on DOM change     | Yes                           | No (if static)                |
| Returned by               | getElementsByTagName()        | querySelectorAll()            |
|                           | getElementsByClassName()      | childNodes                    |
|                           | document.forms/images         |                               |
| CSS selector support      | No                            | Yes                           |
| forEach() support         | No (most browsers)            | Yes                           |
| Index access [0]          | Yes                           | Yes                           |
| length property           | Yes                           | Yes                           |
| Named access (name/id)    | Yes                           | No                            |
| Typical usage today       | Legacy / limited use          | Modern / preferred            |
| Predictability            | Lower (side effects possible) | Higher (safe & predictable)   |
+---------------------------+-------------------------------+-------------------------------+
