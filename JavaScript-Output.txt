1. JavaScript display possibilities
- JavaScript can 'display' data in different ways:
    + Writing into an HTML element, using innerHTML (có thể thay đổi thẻ - tag của phần tử HTML trước đó) or innerText (chỉ thay đổi content của thẻ)
    + Writing into the HTML output using document.write()
    + Writing into an alert bõ, using window.alert()
    + Writing into the browser console, using console.log()

1.1 Using innerHTML
- Để truy cập vào 1 phần tử HTML, có thể truy cập phần tử bằng cách sử dụng 'id' của phần tử thông qua 'document.getElementById(id)
--> Sau đó có thể dùng thuộc tính innerHTML để thay đổi phần tử HTML đó
- Ví dụ:
    <!DOCTYPE html>
    <html>
    <body>

    <h1>My First Web Page</h1>
    <p>My First Paragraph</p>

    <p id="demo"></p>

    <script>
    document.getElementById("demo").innerHTML = "<h2>Hello World</h2>";
    </script>

    </body>
    </html>
- Note: Việc sử dụng innerHTML để thay đổi 1 phần tử HTML là cách phổ biến nhất

1.2 Using innerText
- Để truy cập 1 phần tử HTML, sử dụng 'document.getElementById(id) 
--> Có thể sử dụng thuộc tính innerText để thay đổi content của 1 phần tử HTML
- Ví dụ: 
    <!DOCTYPE html>
    <html>
    <body>

    <h1>My First Web Page</h1>
    <p>My First Paragraph</p>

    <p id="demo"></p>

    <script>
    document.getElementById("demo").innerText = "Hello World";
    </script>

    </body>
    </html>
- Note: chỉ thay đổi content chứ không thay đổi được thẻ tag của phần tử HTML đó --> chỉ dùng khi muốn thay đổi phần content của phần tử bằng plain text

1.3 Using document.write()
- Dùng cho testing --> thuận tiện để sử dụng document.write()
- Ví dụ: 
    + HTML ban đầu:
        <body>
            <h1>Trang Gốc</h1>
        </body>
    + JavaScript chạy sau khi trang hoàn tất:
       setTimeout(() => {
            document.write("<h2>Nội dung mới</h2>");
        }, 2000); 
- Note: 
    + document.write() được thời kế cho HTML bản rất cũ, documet.write() được dùng khi trang còn đang load (trong quá trình parsing HTML). Trong giai đoạn này, browser đang xây dựng DOM, nên document.write() chỉ đơn giản là chèn thêm nội dung vào HTML đang được parse.
    + Nhưng sau khi trang đã load xong (DOMContentLoaded hoặc window.onload). Browser đóng stream HTML lại --> nghĩa là không chấp nhận ghi thêm nội dung vào luồng HTML ban đầu. Nếu mình dùng document.write() sau khi trang đã load thì browser có thể hiểu rằng có phần tử HTML mới (document.write(<p>Đây chỉ là ví dụ</p>)) thì browser sẽ render lại các phần tử HTML lại từ đầu ==> chỉ xuất hiện phần nằm trong document.write()

1.4 Using window.alert()
- Cú pháp đúng: window.alert()
--> Nhưng vì window là 1 object có scope là global. Điều này có nghĩa là các method không khai báo object sẽ được mặc định là của object window, nên là có thể bỏ tên của object ra luôn
--> alert(): ĐƯỢC CHẤP NHẬN 

1.5 Using console.log()
- Dùng cho debugging
--> Cái này quá quen rồi nên khỏi đề cập 

1.6 Using window.print()
- JavaScript không có sẵn:
    + Đối tượng print
    + Phương thức print()
    + API để in ra máy in giống như các ngôn ngữ lập trình khác như Java, C, C++,...
--> Lý do là vì JavaScript chạy trong trình duyệt thường bị giới hạn quyền truy cập phần cứng bảo mặt --> không cho phép truy cập trực tiếp máy in 
- JavaScript không thể truy cập: 
    + Máy in
    + Máy quét
    + Loa/ micro
    + Webcam (chỉ được truy cập qua API đặc biệt và phải xin quyền)
    + Thiết bị cổng USB
    + File hệ thống (phải qua quyền của user)
--> Vì lý do bảo mật, trình duyệt chặn JavaScript không cho tương tác với thiết bị vật lý 
- Dù không có API máy in, JavaScript có 1 ngoại lệ duy nhất:
window.print()
- Đây là phương thức của window.
- Chức năng:
    + Mở hộp thoại in (print dialog) của trình duyệt.
    + In toàn bộ nội dung trang hiện tại
    + In theo CSS được cấu hình
--> JavaScript không điều khiển máy in trực tiếp -> chỉ gửi lệnh để browser mở hộp thoại in lên
